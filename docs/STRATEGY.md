## 2. なぜ勝てないのか — 構造的問題の分析

### 問題A: モデルが「ランキング精度」を最適化しているが「収益」を最適化していない

LambdaRank は「1着艇を正しく予測する順序」を学習する。  
しかし競艇で利益を出すには **「市場（オッズ）が過小評価している組み合わせ」** を見つける必要がある。

```
例:
  モデル: 1号艇 1着確率 60%（正しい）
  オッズ: 1-2-3 = 4.0倍（= 市場の暗示確率 25%）
  
  EV = 0.6 × 4.0 - 1 = +1.4（+140%）  ← これが理想
  
  実際: 人気レースでは1-2-3 = 1.5倍が多く
  EV = 0.6 × 1.5 - 1 = -0.1（-10%）  ← 負け
```

**問題の本質: 強い艇を当てることと、オッズに対して優位性を持つことは別の話。**

### 問題B: EV計算の精度が低い

EV = 予測確率 × オッズ − 1 の「予測確率」精度が不十分。

- Calibration で Brier スコアは改善したが、**3連単の結合確率**（Plackett-Luce）の精度は未検証
- Plackett-Luce モデルは「独立性仮定」を置いているが、実際には **インコース有利・モーター差** などの相関がある

### 問題C: 学習データに「稼げる情報」が欠けている

現在の特徴量はすべて **出走表から取得できる公開情報**。  
市場参加者（他の投票者）も同じ情報を持っており、  
**情報優位性がない → モデルがオッズを上回る確率が低い**。

| 現在の特徴量 | 問題 |
|-------------|------|
| 全国勝率・当地勝率 | 誰でも知っている → オッズに織り込み済み |
| モーター・ボート2連対率 | 同上 |
| 展示タイム | 競艇新聞にも掲載 → 織り込み済み |
| 今節平均ST | 比較的マイナーだが、ST中心レースは専門家も注目 |

### 問題D: サンプル数・テラ銭の壁

3連単のテラ銭は **約25%**。  
EV がプラスになるには、モデルが **市場より25%以上正確** でなければならない。  
これは非常に高いハードルであり、わずかな優位性では吸収されてしまう。

## 3. 勝つための戦略 — 3つのアプローチ

### アプローチ1: 「市場が間違えやすい場面」に絞る（最優先）

全レースに賭けるのではなく、**市場（オッズ）が系統的に間違えやすいパターン**を探す。

**具体的な絞り込み条件の仮説:**

| 条件 | 根拠 |
|------|------|
| 1号艇が人気薄のレース | 競艇ファンは1号艇を過大評価する傾向 → 2号艇以下が割安になりやすい |
| 展示タイムで逆転がある | 番組表では弱く見えるが展示で強い艇 → 市場への反映が遅れる |
| 荒れやすい気象条件（波高高・強風） | 実力差が縮まる → 人気薄に価値が生まれる |
| 後半レース（9R以降）| 前半との比較でオッズ形成が甘くなることがある |
| 地方開催・マイナー場 | 投票額が少なく市場効率が低い可能性 |

**実装方法: レース選択フィルタ**

全レースに賭けず、スコアリングモデルでレースを選別する。

### アプローチ2: モデルに「市場情報」を加える

**オッズ自体を特徴量として学習させる。**

```
現在: 出走表特徴量 → スコア → EV計算
改善: 出走表特徴量 + オッズ暗示確率 → スコア → EV計算
```

オッズの暗示確率 = `1 / (odds × 1.25)`（テラ銭補正）

これを艇ごとの特徴量として加えることで、  
**「市場が過小評価している艇」を直接学習できる。**

ただし注意点:
- 学習時もオッズが必要になる（締切前オッズの過去データが必要）
- スクレイパーで過去オッズを遡って取得するのは困難

**現実的な代替案: 暗示確率との乖離を特徴量にする**

```python
# 暗示1着確率（マーケットの予想）
implied_1st_prob = 1 / (odds_1st * 1.25)

# モデル予測との乖離
model_vs_market = model_prob_1st - implied_1st_prob
# これが大きい艇 = モデルが市場より高く評価している艇
```

### アプローチ3: 損失関数を「収益」に直結させる

現在の LambdaRank は NDCG（ランキング精度）を最適化する。  
**収益率を直接最適化する損失関数に変更する。**

```
現在: minimize( -NDCG )
改善: minimize( -回収率 ) または minimize( -EV )
```

**具体的な方法:**

    """

#### 方法B: 2段階学習

1. Stage1: LambdaRank でランキングを学習（現状維持）
2. Stage2: Stage1 の予測確率を入力として、**「オッズに対して値がいいか」を2値分類**で学習

## 4. 即効性のある改善施策（短期）

実装コストが低く効果が見込める改善から着手する。

### 施策1: レース選択フィルタの実装

```python
# 現状: 全レースに予測・ベット判断
# 改善: 「荒れそうなレース」「モデル信頼度が高いレース」のみ対象

filters = {
    "min_confidence": 0.30,     # 1着予測艇の確率が30%以上
    "min_wave": 5,              # 波高5cm以上（荒れる条件）
    "exclude_1st_odds_under": 2.0,  # 1号艇が2倍未満の鉄板レースは除外
}
```

### 施策2: EV計算の精度向上

現在の Plackett-Luce は独立性を仮定しているが、  
**条件付き確率をより正確に推定する。**

```
1着確率: p1, p2, ..., p6
現在: P(1-2-3) = p1 × p2/(1-p1) × p3/(1-p1-p2)   ← 独立性仮定
改善: P(2着|1着=1) を直接学習する条件付きモデルを追加
```

### 施策3: バックテストによる最適パラメータ探索

現在の `min_edge=0.05` `kelly_frac=0.25` がベストか検証されていない。

```python
# バックテストで最適な閾値を探索
for min_edge in [0.0, 0.05, 0.10, 0.15, 0.20]:
    for kelly_frac in [0.10, 0.25, 0.50]:
        roi = backtest(min_edge=min_edge, kelly_frac=kelly_frac)
        print(f"min_edge={min_edge}  kelly={kelly_frac}  ROI={roi:.1f}%")
```

### 施策4: アンサンブル（複数モデルの多数決）

LightGBM 単体ではなく、複数のモデルの予測を平均する。

```
モデルA: 全期間データで学習（汎化性能重視）
モデルB: 直近2年データで学習（最新傾向を重視）
モデルC: 場ごとに特化したモデル（当地特性を学習）
アンサンブル確率 = (A + B + C) / 3
```

## 6. 現実的な期待値

競艇 AI で収益を出している事例の多くは以下の条件を満たしている:

| 条件 | 本システムの現状 |
|------|----------------|
| 数万レースの学習データ | ✅ 2009〜2024年（約15万レース） |
| リアルタイムオッズ活用 | ✅ 実装済み |
| レース選択（全賭けしない） | ❌ 未実装（最重要改善点） |
| 市場情報の活用 | ❌ オッズを特徴量として学習していない |
| 収益直結の損失関数 | ❌ NDCG（ランキング精度）のまま |

**正直な見解:**

現状のモデルで安定的な黒字は難しい。  
テラ銭 25% を越えるには、公開情報だけでは市場と対等であり、  
**「レース選択」と「オッズを特徴量に加えること」が最も効果が見込める改善**である。

ドライランで1ヶ月データを集めてから、どの施策から着手するかを再判断することを推奨する。
